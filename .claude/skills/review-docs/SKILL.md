# ドキュメントレビュー

指定されたドキュメントをレビュー: $ARGUMENTS

## レビュー手順

### 1. 対象ファイルの読み込み
指定されたファイル、または `docs/` 全体をスキャン

### 2. チェック項目

#### 正確性
- パラメータ名がソースコードと一致しているか
- デフォルト値が正しいか
- コード例が構文的に正しいか

#### 完全性
- frontmatter（sidebar_position）があるか
- 必要なセクションが揃っているか
- IMAGE_NEEDED/VIDEO_NEEDED のプレースホルダーが説明的か

#### スタイル
- 日本語が自然か
- フォーマットが一貫しているか
- コードブロックに言語タグがあるか

### 3. ビルドテスト
```bash
npm run build
```

### 4. レポート出力

```markdown
## レビュー結果

### 問題点
- [ ] [ファイルパス]: [問題の説明]

### 改善提案
- [提案内容]

### 統計
- 確認ファイル数: X
- 問題数: Y
```
